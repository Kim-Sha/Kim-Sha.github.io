## Overview

Personal website / blog for Kim Sha built with Gatsby JS (React), hosted on GitHub Pages. Uses MDX for blog posts, YAML for structured content data, Tailwind CSS for styling, Emotion for CSS-in-JS, and Plotly for interactive charts.

## Commands

```shell
# Start local dev server at http://localhost:8000
gatsby develop

# Build static site to /public
gatsby build

# Serve the built site locally
gatsby serve

# Format all JS/JSON/MD files
npm run format

# Clear Gatsby cache (run when builds break unexpectedly)
gatsby clean

# Deploy to GitHub Pages (builds then pushes /public to main branch)
npm run deploy
```

GraphQL explorer is available at `http://localhost:8000/___graphql` during development.

## Architecture

### Data Flow

All site content lives in `content/` as YAML or MDX files. Gatsby reads these at build time via `gatsby-source-filesystem` + `gatsby-transformer-yaml` + `gatsby-plugin-mdx`, making them available via GraphQL.

- **YAML files** (`profile.yaml`, `work-history.yaml`, `projects.yaml`, `social.yaml`, `devtool.yaml`) ‚Äî structured profile data queried directly by components via `useStaticQuery`
- **MDX files** (`content/blog-posts/**`) ‚Äî blog posts with frontmatter; pages are auto-generated by `gatsby-node.js` using the `src/templates/blog.js` template. Slugs are derived from file paths relative to `blog-posts/`.

### GraphQL Fragments

Reusable GraphQL fragments are defined in `src/types/` (e.g., `ProfileFragment`, `SocialFragment`). These must be imported/co-located with the components that use them.

### Theming

Themes are defined in `src/themes/*.js` (e.g., `ksha-light.js`, `ksha-dark.js`). The active theme is set in two places:
1. `gatsby-config.js` ‚Äî passed to Tailwind's PostCSS config: `require("./src/utils/tailwind.config")("ksha-light")`
2. `src/templates/theme-context.js` ‚Äî provides runtime theme switching via React context (`ThemeProvider`)

### Layout

`src/templates/layout.js` is the root layout wrapper used by all pages. It composes: `Header` + `Sidebar` + `MDXProvider` (with `LazyPlot` registered as a global MDX component) + `Footer`.

### Interactive Plots in Blog Posts

Plotly figures used in MDX blog posts are stored as JSON in the same directory as the post (e.g., `content/blog-posts/nrel/nrel_figures.js`). The `<LazyPlot>` component (`src/components/plots/plotly.js`) is available globally in MDX without imports ‚Äî it lazy-loads `react-plotly.js` to avoid SSR issues.

### Updating Site Content

To update profile info, work history, skills, or links ‚Äî edit the relevant YAML file in `content/`. No code changes needed; Gatsby picks them up automatically.

To add a blog post: create a new directory under `content/blog-posts/` with an `index.mdx` file. The slug will be derived from the directory path.
